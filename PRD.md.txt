# 产品需求文档 (PRD)：轻量化脚手架安全计算软件 (Web端)

## 1. 产品概述
* **产品定位：** 一款面向建筑行业一线技术人员（施工员、方案编制员、总工）的轻量化、Web端脚手架安全验算与计算书生成SaaS软件。
* **核心痛点：** 传统PC端工程软件操作繁琐、学习门槛高、价格昂贵；传统Excel表格计算存在合规风险且无法自动生成图文并茂的Word计算书。
* **核心价值：** 极简的网页表单填报体验，一键生成符合最新国家规范（如JGJ/T 231-2021等）、包含专业力学简图且排版精美的Word格式专项施工方案计算书。

## 2. 商业模式与目标用户
* **目标用户：** 建筑企业基层技术员、方案深化设计师、中小型劳务/租赁公司技术人员（面向C端或小B端）。
* **商业模式：** Freemium（免费增值）+ SaaS订阅制（按次付费、包月、包年会员）。
* **使用场景：** 网页端即开即用，输入参数 -> 实时预览结果 -> 扫码付费 -> 下载标准Word文档。

## 3. 产品架构与演进路线
* **第一阶段 (MVP - 本次开发核心)：** 盘扣式双排外脚手架（落地式）。跑通“前端表单交互 -> 后端力学验算及动态绘图 -> Word模板渲染注入”的全链路。
* **第二阶段 (横向拓展，已做架构预留)：** 盘扣式支撑架系列（梁模板共用立柱、梁底双托梁、托梁兼主梁、板模板）、边梁模板、卸料平台。

## 4. MVP 功能需求说明 (盘扣式双排外脚手架)

### 4.1 用户操作流 (User Flow)
1.  用户访问Web应用，选择“盘扣式双排外脚手架”模块。
2.  系统展示分步表单（带默认经验值），用户按需修改工程参数。
3.  前端实时校验参数合理性（如长细比超限预警、搭设高度预警）。
4.  点击“开始验算”，前端将JSON数据发送至后端API。
5.  后端执行力学计算：
    * 若验算不通过：返回不通过的具体项及建议修改提示。
    * 若验算通过：后端在内存中动态绘制剪力图、弯矩图等配图，并静默渲染Word文档，返回成功状态及计算结果摘要。
6.  用户确认结果，触发支付逻辑（或扣除会员权益），下载生成的`.docx`计算书文件。

### 4.2 前端表单数据结构 (Input Data Schema)
为方便Codex生成表单和接口，划分为以下参数组。所有非文本项均需设置合理的默认值。

* **Group 1: 基本信息 (`basic_info`)**
    * 工程名称、施工部位、编制人、计算日期、脚手架安全等级（I级/II级）、结构重要性系数（1.0/1.1）。
* **Group 2: 架体几何参数 (`geometry_params`)**
    * 立杆纵距 (la)、立杆横距 (lb)、立杆步距 (h)、搭设高度 (Hs)、架体排数（固定为2）、连墙件布置方式（如“一步两跨”）、扫地杆高度、防护栏杆高度。
* **Group 3: 材料与荷载参数 (`material_load_params`)**
    * 立杆钢管型号、水平杆钢管型号（下拉选择，后端需关联物理属性库）。
    * 脚手板类型（定型钢脚手板等）及自重标准值。
    * 密目网/挡脚板自重标准值。
    * 作业层数、施工均布活荷载（如 3kN/㎡）。
    * 基本风压 (w0)、地面粗糙度类别（A/B/C/D）、风荷载体型系数。
* **Group 4: 地基基础参数 (`foundation_params`)**
    * 地基土类型、地基承载力特征值 (fg)、垫板底面积。

### 4.3 后端计算逻辑流转 (Calculation Engine Pipeline)
后端算法模块需严格按照以下顺序实例化并执行计算，并将每一步的关键结果存入上下文中，以备最终注入Word模板。

1.  **横向横杆验算：**
    * 计算静/活荷载产生的线荷载。
    * 进行抗弯强度验算、挠度验算。
    * 计算支座反力，验算盘扣节点抗剪承载力。
2.  **荷载统计：**
    * 计算单立杆静荷载（立杆、横杆、脚手板、围护材料自重）。
    * 计算单立杆施工活荷载。
    * 汇总得出立杆最大轴力设计值 (N)。
3.  **立杆稳定性验算 (核心)：**
    * 计算立杆长细比 (λ)，查表得出稳定系数 (φ)。
    * 计算风荷载产生的弯矩 (Mw)。
    * 组合轴力和弯矩，验算立杆稳定性 (σ ≤ [f])。
4.  **连墙件承载力验算：**
    * 计算风荷载及平面外变形产生的轴向力。
    * 验算连墙件抗压/抗拉强度、扣件抗滑移承载力、螺栓锚固/局部承压。
5.  **底座与地基基础验算：**
    * 可调底座承载力验算。
    * 立杆支承面（地基）承载力验算 (p ≤ kc × fg)。
6.  **动态配图生成 (Diagram Generation)：**
    * 利用上述力学计算的内力数组，绘制受力简图、剪力图、弯矩图及挠度变形图，将图片保存为内存流 (BytesIO)。

### 4.4 输出要求 (Output)
* **渲染技术：** 必须使用Python的 `docxtpl` (DocxTemplate) 库。
* **动态配图插入：** 通过 `docxtpl` 的 `InlineImage` 功能，将后端动态生成的受力简图、剪力图和弯矩图流无缝插入到 Word 模板的对应位置。
* **实现方式：** 本地维护一个排版完美的 `.docx` 模板文件，其中使用 Jinja2 语法（如 `{{ max_stress }}` 或图形占位符 `{{ moment_diagram }}`）。后端计算及绘图完成后，构建一个包含所有数据的上下文对象（Context Dict），渲染生成最终文件。

## 5. 技术架构与选型约束
* **前端框架：** Vue 3 或 React (推荐结合 Element Plus 或 Ant Design，利用其完善的表单验证机制)。
* **后端框架：** Python + FastAPI (极轻量、原生支持异步、非常适合处理科学计算和I/O密集型的文档生成)。
* **图形绘制库：** `matplotlib`（用于在后端纯代码生成工程力学所需的受力简图、剪力图、弯矩图等二维图表）。
* **数据库：** PostgreSQL 或 MySQL。
* **架构解耦设计：**
    * **数据库扩展性：** 主表 `projects` 存储通用字段，所有具体业务参数使用 `JSON` 类型的 `params_data` 字段存储，以适应未来梁板模块截然不同的参数结构。
    * **计算内核设计模式：** 采用工厂模式或策略模式。定义基类 `BaseCalculator`，当前MVP实现子类 `ExteriorScaffoldCalculator`。前端请求携带 `scaffold_type`，后端路由自动分发至对应的计算类，并调用该类的计算、绘图及报告生成方法。
